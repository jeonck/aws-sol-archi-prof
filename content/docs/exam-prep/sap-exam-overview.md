AWS Certified Solutions Architect - Professional(SAP-C02)은 AWS 자격증 중 가장 어려운 시험으로 꼽힙니다. SAA-C03이 "이 워크로드를 어떻게 설계할 것인가"를 묻는다면, SAP-C02는 "이미 존재하는 복잡한 조직과 레거시 환경에서, 여러 제약 조건을 동시에 만족하는 최선의 선택은 무엇인가"를 묻습니다.

## 시험 구조

| 항목 | 내용 |
|---|---|
| 시험 코드 | SAP-C02 |
| 시험 시간 | 180분 |
| 문항 수 | 75문제 (객관식 + 복수 응답) |
| 합격 기준 | 1000점 만점 중 약 750점 (영역별 가중 채점) |
| 응시료 | $300 USD |
| 시험 형식 | Pearson VUE 또는 PSI를 통한 시험 센터 응시 또는 온라인 proctoring |
| 유효 기간 | 합격 후 3년 |

응시료가 SAA의 두 배($300)이고 시험 시간도 50분 더 길지만, 문항당 평균 풀이 시간은 SAA(문제당 약 2분)보다 SAP(문제당 약 2.4분)가 조금 더 여유롭습니다. 문제 자체가 길고 복잡해서 읽는 시간이 오래 걸리기 때문입니다.

## SAA와의 핵심 차이 — 시나리오와 Trade-off

SAP-C02 문제의 전형적인 구조는 다음과 같습니다.

> "한 회사가 현재 [복잡한 기존 환경]을 운영 중이다. [여러 제약 조건: 비용, 규정 준수, 가용 시간, 마이그레이션 기한 등]이 있다. 다음 중 이 요구 사항을 **가장 적은 운영 부담으로** 만족하는 솔루션은?"

이런 문제에서는 보통 4개의 선택지 중 2~3개가 "기술적으로는 작동하는" 답입니다. SAA처럼 명백히 틀린 선택지를 제거하는 것만으로는 정답을 좁히기 어렵고, 문제에 명시된 제약 조건(비용 효율, 운영 단순성, 보안, 마이그레이션 기한 등) 중 무엇이 우선인지를 파악해 trade-off를 판단해야 합니다.

대표적인 차이를 정리하면 다음과 같습니다.

| 구분 | SAA-C03 | SAP-C02 |
|---|---|---|
| 문제 길이 | 짧음, 단일 요구사항 위주 | 길고, 여러 제약 조건이 동시에 제시됨 |
| 정답 패턴 | 명백히 틀린 선택지 제거로 좁혀짐 | 여러 개가 "맞는" 답처럼 보임, 우선순위 판단 필요 |
| 다루는 범위 | 단일 워크로드 설계 | 조직 전체(멀티 계정), 마이그레이션, 레거시 통합 |
| 요구하는 깊이 | 서비스의 기능과 사용 시점 이해 | 서비스 간 조합, 비용·운영·보안의 동시 최적화 |

이 차이 때문에 SAP-C02는 단순 암기나 서비스 스펙 학습으로는 통과하기 어렵습니다. 실제로 여러 서비스를 조합해 마이그레이션 계획을 세워보거나, 멀티 계정 거버넌스를 구성해본 경험이 문제 해석 속도를 크게 높여줍니다.

{{< callout type="warning" >}}
SAP-C02 응시를 고려하고 있다면, SAA 합격 직후 바로 도전하기보다 실무에서 최소 6개월~1년 이상 AWS 아키텍처 설계나 운영 경험을 쌓은 뒤 응시하는 것이 일반적으로 권장됩니다. 경험이 부족하다면 본 로드맵의 **[③ Solutions Architect Professional](../../sap/)** 섹션의 각 주제를 콘솔에서 직접 구성해보며 경험을 보완하세요.
{{< /callout >}}

시험이 실제로 무엇을 도메인별로 평가하는지, 그리고 각 도메인이 이 로드맵의 어느 페이지와 연결되는지는 **[SAP-C02 시험 청사진](../sap-exam-domains/)** 에서 표와 다이어그램으로 정리했습니다.

## 실전 감각 채우기

SAP-C02 학습의 마지막 단계는 **[도메인 3: 지속적인 개선](../../sap/domain3-continuous-improvement/)** 에서 다루는 Well-Architected Tool 기반 정기 리뷰 프로세스를 직접 경험해보는 것입니다. 실제 워크로드(또는 실습용 아키텍처)를 Well-Architected Tool에 등록하고 리뷰를 진행해보면, 시험에서 자주 등장하는 "이 설계의 위험 요소를 식별하라" 유형의 문제에 훨씬 익숙해집니다. Practice Exam 활용법은 **[Practice Exam 전략](../practice-exam-strategy/)** 을 참고하세요.
