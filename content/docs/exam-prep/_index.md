지금까지의 학습이 머릿속에서 어떻게 정리되어 있는지는 시험장에 들어가기 전까지 정확히 알 수 없습니다. 이 섹션은 SAA-C03과 SAP-C02 두 시험의 실제 구조(시간, 문항 수, 합격 기준, 출제 영역)를 먼저 파악하고, Practice Exam을 어떻게 활용해야 합격 가능성을 신뢰성 있게 가늠할 수 있는지, 그리고 어떤 자료를 어떤 순서로 써야 시간을 낭비하지 않는지를 다룹니다.

{{< callout type="warning" >}}
시험 전략은 "시험 막바지에 한 번 읽는 페이지"가 아닙니다. SAA나 SAP 학습을 시작하는 시점에 먼저 이 섹션을 한 번 읽고 시험의 전체 그림(시간 배분, 출제 비중, 합격 기준)을 파악한 뒤, 학습 중반부터는 매주 Practice Exam 루틴을 실제로 가동해야 합니다.
{{< /callout >}}

## 이 섹션에서 다루는 내용

1. **[SAA-C03 시험 개요](saa-exam-overview)** — 시험 구조, 합격 기준, 등록 방법, 출제 영역 비중
2. **[SAP-C02 시험 개요](sap-exam-overview)** — 시험 구조와 SAA 대비 난이도·문제 유형 차이
3. **[SAP-C02 시험 청사진](sap-exam-domains)** — 공식 Exam Guide의 4개 도메인·작업(Task)·범위 내 서비스를 도식으로 재구성
4. **[Practice Exam 전략](practice-exam-strategy)** — 매주 루틴, 오답 분석법, 시나리오 문제 접근법
5. **[추천 학습 자료](recommended-resources)** — Skill Builder, Udemy, Tutorials Dojo, 공식 백서, re:Invent
6. **[실전 문제 풀이 기술](real-exam-tactics)** — 키워드 매핑, 긴 지문 스키밍, 2~3회독 전략
7. **[SAP-C02 샘플 문제 10선](sap-sample-questions)** — 실제 시험과 동일한 유형의 샘플 문제 10선 및 상세 해설
8. **[AWS 아키텍처 핵심 원리와 제약](architecture-principles)** — AWS 설계 철학 및 자원 할당량(Quota) 제한 등 핵심 설계 원리
