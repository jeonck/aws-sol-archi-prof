AWS가 공식으로 제공하는 학습 플랫폼은 대부분 무료이거나 매우 저렴하며, 실제 콘솔 환경에서 손을 움직이며 배울 수 있다는 공통점이 있습니다. 이론 학습과 병행해서 꾸준히 활용하세요.

## AWS Skill Builder

- **링크**: https://skillbuilder.aws
- **설명**: AWS가 운영하는 공식 무료 학습 플랫폼입니다. 디지털 배지 과정, 학습 경로(Learning Plan), 그리고 SAA-C03/SAP-C02용 **Exam Prep Plan**을 제공합니다. Exam Prep Plan은 시험에 나오는 도메인별로 학습 콘텐츠와 연습 문제를 자동으로 묶어주므로, 시험 준비 단계에서 특히 유용합니다.
- **시작 방법**: AWS 계정(또는 별도의 Skill Builder 계정)으로 로그인한 뒤, 검색창에 "Exam Prep" 또는 응시할 자격증 이름을 입력해 학습 경로를 찾습니다. 일부 고급 콘텐츠는 유료 구독(Individual 플랜)이 필요합니다.

## AWS Cloud Quest

- **링크**: https://aws.amazon.com/training/digital/aws-cloud-quest/
- **설명**: 3D 게임 형태로 AWS 개념을 학습하는 역할극 기반 콘텐츠입니다. Cloud Practitioner, Solutions Architect 역할(role-playing) 모드가 있어, 가상의 시나리오 속에서 실제 AWS 리소스를 생성·연결하며 미션을 클리어합니다. 딱딱한 문서보다 진입 장벽이 낮아 입문자에게 추천됩니다.
- **시작 방법**: Skill Builder 계정으로 로그인 후 Cloud Quest 메뉴에서 원하는 역할(Practitioner/Solutions Architect)을 선택해 바로 플레이할 수 있습니다.

## AWS Builder Labs

- **링크**: https://aws.amazon.com/training/digital/aws-builder-labs/
- **설명**: 실제 AWS 콘솔과 리소스를 사용해 정해진 시나리오를 따라 실습하는 핸즈온 랩 모음입니다. 가이드를 따라가며 EC2, S3, VPC 등 핵심 서비스를 직접 구성해보고, 랩이 끝나면 리소스가 자동으로 정리되어 비용 부담이 적습니다.
- **시작 방법**: Skill Builder 내 Builder Labs 카탈로그에서 주제를 검색하고, 크레딧(무료 제공량 또는 소액 결제)을 사용해 랩을 시작합니다.

## AWS Jam

- **링크**: https://aws.amazon.com/blogs/training-and-certification/getting-started-with-aws-jam/
- **설명**: 팀 단위로 진행하는 실전형 챌린지 이벤트입니다. 정해진 시나리오 속 장애나 문제를 제한 시간 안에 해결하며 점수를 얻는 게임화된 형태로, re:Invent나 AWS Summit 같은 행사에서 자주 운영되고 온라인으로도 참가할 수 있습니다. 실무에서 마주칠 법한 트러블슈팅 감각을 키우는 데 좋습니다.
- **시작 방법**: AWS 행사 일정이나 공식 블로그에서 Jam 이벤트 참가 링크를 확인하거나, 팀/조직 단위 Jam 세션을 신청해 참여합니다.

## AWS Free Tier

- **링크**: https://aws.amazon.com/free
- **설명**: 신규 계정 생성 시 12개월간 일정량의 EC2, S3, RDS 등 핵심 서비스를 무료로 사용할 수 있고, 일부 서비스는 항상 무료(Always Free) 한도를 제공합니다. 로드맵의 모든 Hands-on 실습은 기본적으로 이 Free Tier 환경을 전제로 합니다.
- **시작 방법**: 계정 생성 직후 **AWS Budgets**에서 예산 알림(Budget Alert)을 설정해 일정 금액(예: $1, $5)을 초과하면 이메일로 통보받도록 구성하세요. Free Tier 한도를 넘기는 실수를 조기에 방지할 수 있습니다. 자세한 계정 설정 절차는 [AWS 기초 — Free Tier 계정 설정](../../docs/fundamentals/free-tier-account-setup/)을 참고하세요.
